Web Workers

Single threading in JS bringing you down?

Then this is probably the answer to your problem.

Some Code

"Main thread"


var worker = new Worker('/js/worker.js');
worker.addEventListener('message', function(e) {
  console.log(e.data);
}, false);
worker.postMessage({'cmd':'start'});
worker.postMessage({'cmd':'stop'});
worker.postMessage({'cmd':'get'});

"Worker thread"


self.addEventListener('message', function(e) {
  self.postMessage({'msg':'WORKER STARTED'});
}, false);

When do you need this?

  • Worker scripts do not block UI scripts
  • Long running worker scripts do not interrupt UI thread.
  • Expensive tasks go to workers
  • Tired of mimicking concurrency with setInterval()/setTimeout()/XMLHttpRequest
  • Safari,Chrome,Firefox,Opera

function gotStream(stream) {
  var audioContext = 'AudioContext' in window ? new AudioContext() :
    (('webkitAudioContext' in window) ? new webkitAudioContext() : null);
  if (audioContext) {
    var mediaStreamSource = audioContext.createMediaStreamSource(stream);
    mediaStreamSource.connect(audioContext.destination);
  } else {
    // ...
  }
}

navigator.webkitGetUserMedia({audio:true}, gotStream);

When do you need this?

  1. Real time communication w/o plugins.
  2. Works with Chrome/Firefox
