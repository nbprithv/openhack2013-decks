Web Workers

Single threading bringing you down?

Some Code

"Main thread"


var worker = new Worker('/js/worker.js');
worker.addEventListener('message', function(e) {
  console.log(e.data);
}, false);
worker.postMessage({'cmd':'start'});
worker.postMessage({'cmd':'stop'});
worker.postMessage({'cmd':'get'});

"Worker thread"


self.addEventListener('message', function(e) {
  self.postMessage({'msg':'WORKER STARTED'});
}, false);

When do you use this?

  • Worker scripts do not block UI scripts
  • Long running worker scripts do not interrupt UI thread.
  • Expensive tasks go to workers
  • Tired of mimicking concurrency with setInterval()/setTimeout()/XMLHttpRequest
  • Safari,Chrome,Firefox,Opera

Webkit Desktop Notification

Less annoying pop-ups.

Some Code


function notify() {
  var havePermission = window.webkitNotifications.checkPermission();
  if (havePermission == 0) {
    // 0 is PERMISSION_ALLOWED
    var notification = window.webkitNotifications.createNotification(
      'http://localhost:3000/notifications',
      'Chrome notification!',
      'Ola!'
    );

    notification.show();
  } else {
      window.webkitNotifications.requestPermission();
  }
}

When do you use this?

  1. Safari
  2. Chrome
